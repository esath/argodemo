△ ArgoCD on OpenShift

CI/CD Demo — GitOps Deployment

v1.1.0  |  Deployed via ArgoCD

🔄 How This Demo Works

  1. A developer pushes a code change to this Git repository.
  2. The GitHub Actions CI workflow builds a new container image and pushes it to the container registry, then updates the image tag in k8s/deployment.yaml.
  3. ArgoCD detects change in the Git repository and automatically syncs the new manifests to the OpenShift cluster.
  4. OpenShift rolls out the new pod — zero-downtime deployment.
  5. This page refreshes and you see the updated version.

✏️ What to Modify in This Demo

File What to Change Effect
app/index.html Edit the version badge, heading, or any visible text New container image built & deployed
k8s/deployment.yaml Change replicas, resource limits, or env vars OpenShift scales / reconfigures pods
k8s/route.yaml Change the hostname or TLS settings Route / ingress updated
argocd/application.yaml Change sync policy, destination namespace, or source repo ArgoCD behavior updated
.github/workflows/ci.yaml Add build steps, linting, or push to a different registry CI pipeline customised

🚀 Quick-Start: Trigger a Deployment

  1. Edit app/index.html — for example, change the version badge from v1.0.0 to v1.0.1. <span class="badge">v1.0.1 | Deployed via ArgoCD</span>
  2. Commit and push: git add app/index.html git commit -m "demo: bump version to v1.0.1" git push
  3. Watch GitHub Actions build the image and update k8s/deployment.yaml.
  4. Open the ArgoCD UI — the app will show OutOfSync and then automatically sync within the configured interval.
  5. Reload this page — you should see the new version.
